01 / How This Works

Claude isn't one thing. There are several versions of it, and most people are using the one that forgets them the moment the tab closes.

Not Recommended Claude Chat (claude.ai)

The browser chat you've probably already used. Every new conversation starts blank. It doesn't know you, your business, or your voice unless you re-explain everything each time. Fine for quick one-offs. Not a place to build a business system.

Recommended Claude Code in VS Code

This is what this guide sets up. VS Code is a free workspace made by Microsoft. Think of it as a smart filing cabinet where Claude lives alongside your documents. Claude can read your files, create new ones, and run tasks for you while you watch and approve. It reads your foundation files at the start of every session, so it always knows who you are.

What "foundation files" means

Four documents that teach Claude your voice, your audience, your business goals, and the writing patterns to avoid. Claude reads them before it does anything else, every session. You build these by having one guided conversation with Claude, not by filling out a form. Your job is Sections 02 and 03, then one paste.

02 / Accounts and Apps (Your Part)

Three things Claude cannot do for you. Budget 20 minutes.

1

Create a Claude account

Go to claude.ai, sign up with your Google email, and upgrade to Claude Pro. Pro is $20/month and required for Claude Code. Without it you'll hit usage limits within minutes.

2

Download VS Code

Download it at code.visualstudio.com/download and follow the instructions for your computer.

Mac
  1. Click the .dmg Apple Silicon button (or Intel if your Mac is older than 2020)
  2. Open the downloaded file from your Downloads folder
  3. Drag the VS Code icon into your Applications folder
  4. Open VS Code from Applications or Launchpad
  5. If you see a security warning, go to System Settings → Privacy & Security and click Open Anyway
Windows (PC)
  1. Click the Windows User Installer x64 button
  2. Run the downloaded .exe file and click Next through the setup
  3. Important: on the "Additional Tasks" screen, check Add to PATH
  4. Also check Add "Open with Code" action
  5. Click Install, then Finish, then open VS Code from your Start menu
Windows users: also install Git

Git is a tool Claude Code uses to manage files, and Windows doesn't include it. Go to git-scm.com/download/win, run the installer with all the default settings, and restart VS Code afterward. Mac users skip this.

3

Install the Claude Code extension

Inside VS Code: click the Extensions icon in the left sidebar (four small squares), search for Claude Code, find the one by Anthropic, and click Install. A Claude icon appears in your left sidebar. Click it and sign in with the same account you used on claude.ai.

03 / Your Claude Folder

One folder, two clicks. This is the last setup step you do by hand.

1

Create a folder called Claude

On your computer, make a new empty folder named Claude. Your Documents folder or Desktop both work. Everything Claude builds will live here.

2

Open it in VS Code

In VS Code, go to File → Open Folder and pick your Claude folder. The left panel now shows your (empty) folder. That's correct. Claude fills it in the next step.

Keep this folder open every time you work

VS Code remembers your last opened folder, so it usually reopens in the right place. If Claude ever says it can't find a file, the wrong folder is open. File → Open Folder → Claude fixes it.

04 / The One Thing You Paste

Click the Claude icon in the sidebar, start a conversation, and paste this whole box. That's the entire install.

Paste this into Claude (use the Copy button)Set up my Claude workspace. Follow these steps exactly. 1. Download my team's setup system by running these two commands: curl -fsSL "https://how-to-elevate.github.io/elevate-claude-code-setup/skills/setup/SKILL.md" --create-dirs -o ".claude/skills/setup/SKILL.md" curl -fsSL "https://how-to-elevate.github.io/elevate-claude-code-setup/skills/build-icp/SKILL.md" --create-dirs -o ".claude/skills/build-icp/SKILL.md" 2. Read the file .claude/skills/setup/SKILL.md and follow it exactly, step by step, starting now.
Claude will ask your permission as it works

The first few times Claude downloads a file or creates a folder, a small prompt appears asking you to allow it. Click Yes or Allow. That's the approval system working, not something going wrong. You are always in charge of what Claude does on your computer.

Life happened and you closed your laptop mid-setup?

Paste the same box again in a new conversation. Claude keeps a progress file, sees exactly where you stopped, and picks up from there. Nothing is lost and nothing repeats.

05 / What Happens Next

From here, setup is a conversation. Claude asks. You answer. No more pasting.

Here's the shape of what Claude will walk you through, so nothing surprises you:

  • It looks before it asks. Claude first checks what it can already learn about you, and asks if you're coming from ChatGPT or Claude on the web so it can bring your history over instead of making you repeat yourself.
  • It interviews you, one question at a time. Who you are, how you work, your goals, and the writing patterns you can't stand. Answer like you're talking to a friend. Honest and specific beats polished.
  • The ideal client interview goes deep. This is the longest part and the most valuable. Early on, Claude asks whether your buyer is buying for their own life (someone investing in their own family, health, or future) or for their job (a manager choosing a vendor for their company). Answer honestly. The whole interview reshapes itself around your answer.
  • You approve every file before it's saved. Claude drafts, shows you, and only saves when you say yes. Your edits always win.
  • It ends with proof. Claude writes a few lines in your voice, to your person, and asks one question: does this sound like you?
Talk instead of type

A voice-to-text tool that types as you speak, in any app, makes the interview go twice as fast. If you'd rather talk through your answers than type them, look into one before you start (Wispr Flow is a free option). Five minutes to set up, worth it for this step alone.

What you'll have when it's done

Four foundation files (your voice, your ideal client, your company, your anti-AI writing rules), a full ideal-client dossier with your buyer's real language, a CLAUDE.md instruction file that loads all of it automatically every session, and a built-in /weekly-check-in command that keeps it all current. You never explain yourself to Claude again.

06 / Your Daily Workflow

Every new conversation starts with Claude already knowing you. Just say what you're working on.

How to start a session

For a new projectI'm starting a new project called [name]. Create a folder for it inside my PROJECTS folder, then let's begin.
For an existing projectOpen the [project name] folder in PROJECTS. Here's what we're working on today: [brief]
For a one-off piece of contentI need [type of content] for [purpose]. It's for [audience description]. Here are the key points to include: [list]
For strategy workI want to think through [topic]. Read my foundation files first, then let's have a working conversation. I'll share my thinking and I want pushback, suggestions, and clarity.

How to close a session

Save session notesSummarize what we decided and built today. Save a session-notes.md file in the [project name] folder so we have a record.
Draft the next openerWrite an opening prompt I can paste at the start of our next session to pick up exactly where we left off.

Once a week

Type /weekly-check-in at the start of any session. Claude reviews your foundation files, flags what's gone stale, and asks what changed in your business. Ten minutes. Setup already installed this for you; there's nothing to create.

07 / Troubleshooting

Check here before assuming anything is broken.

The Problem What's Causing It How to Fix It
Setup got interrupted Closed laptop, lost connection, ended the chat Paste the Section 04 box again in a new conversation. It resumes where you stopped.
The download step fails A hiccup reaching the hosting site Tell Claude: "Try the downloads again, and if they still fail, use raw.githubusercontent.com/how-to-elevate/elevate-claude-code-setup/main/ instead of the how-to-elevate.github.io address."
Claude keeps asking permission Nothing. That's the approval system. Click Yes or Allow. It settles down after the first few.
Writing sounds generic anti-ai-writing-style.md is too vague Run /weekly-check-in and add specific words and patterns to ban
Claude asks what it already knows Wrong folder is open, so CLAUDE.md isn't being read File → Open Folder → select your Claude folder
File not found errors Same cause: wrong folder open Same fix: File → Open Folder → Claude
Responses getting slow or confused The conversation has gotten too long Start a fresh conversation. Long threads degrade quality.
Unsure if a new session is using the right folder Starting a session with "Local" vs "Web" options, and not sure which applies Choose Local. That's the one that sees the files on your own computer. Also glance at the top of the file list on the left, it should show "CLAUDE."
Keep these habits in place

One task per conversation. Start fresh when you switch focus.

Your CLAUDE.md runs every session. Edits take effect in the very next conversation.

Update foundation files monthly at minimum. Stale files produce stale output.

You are always in authority. Claude proposes. You decide.

08 / When You're Ready for More

Everything below is optional, and none of it needs a guide. You just ask Claude.

Bring in existing brand assets

If you have past campaigns, brand guidelines, or scripts, drop them in a folder called BRAND inside your Claude folder, then paste:

Asset auditI've added existing brand assets to my BRAND folder. Review everything in there and tell me: what's worth keeping, what patterns you notice in my voice and structure, what should be added to my foundation files, and what I should retire.

Use your foundation on your phone

Mobile accessI want to use my foundation files in the Claude mobile app. Walk me through setting up a Claude Project on claude.ai with my four ABOUT ME files uploaded, step by step.

Back up your workspace

A free GitHub account backs up your whole workspace so it's never lost, even if your computer is. Create the account at github.com with the same email you used for Claude, then paste:

GitHub backupFirst review all files in my workspace and flag anything sensitive that shouldn't be stored online, and exclude it. Then connect my Claude folder to a new private GitHub repository and push the contents. Walk me through each step; I've never used GitHub.

Build your first automation

Automation kickoffI want to build an automation for [describe the task]. Before you build anything: 1. Read my CLAUDE.md and foundation files 2. Create a new folder inside PROJECTS called [automation name] 3. Draft a plan: what it will do, what it needs from me, what it produces, and where it saves output 4. Present the plan for my review before building I'll approve the plan before we proceed.
One rule for anything involving passwords or API keys

Never paste a password or API key into the chat. If an automation needs a credential, tell Claude and it will help you set up a secure way to store it. If a key ever lands in a conversation by accident, treat it as exposed and replace it.
